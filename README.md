# CI/CD Pipeline Security Mini Project

## Objective
The objective of this project is to demonstrate a basic CI/CD pipeline using GitHub Actions and implement simple security practices.

## Project Structure

- src/
  - Contains the website files (HTML, CSS, JavaScript).

- .github/workflows/
  - Contains the GitHub Actions workflow (ci.yml) that automatically runs whenever code is pushed to GitHub.

- Dockerfile
  - Used to create a Docker image for the application.

- README.md
  - Provides documentation about the project.

- .gitignore
  - Prevents unnecessary files from being uploaded to GitHub.

## Technologies Used

- HTML
- CSS
- JavaScript
- Git
- GitHub
- GitHub Actions
- Docker

## Security Features

- Automated GitHub Actions workflow
- CodeQL static code analysis
- Version control using Git
- Secure source code management

## Workflow

1. Write code.
2. Commit changes using Git.
3. Push code to GitHub.
4. GitHub Actions automatically runs the CI pipeline.
5. CodeQL scans the code for security vulnerabilities.

## Outcome

The project demonstrates how Continuous Integration (CI) automates testing and security checks, helping developers identify issues before deployment.
